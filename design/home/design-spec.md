# VTCC 微信原生小程序：首页设计交付规范

> 基准文件：`vtcc-home.html`（最终确认稿）  
> 适用范围：仅 VTCC 微信小程序首页  
> 开发原则：原生小程序实现必须复现当前视觉、尺寸、信息层级和状态行为，不对页面重新设计。

## 1. 页面结构

页面采用单列纵向滚动，基准视口宽度为 `375px`。主内容左右留白 `16px`，底部为固定导航。

| 顺序 | 模块 | 对应结构 | 说明 |
|---:|---|---|---|
| 1 | 系统状态栏 | `.statusbar` | 高 24px，展示时间及系统状态示意 |
| 2 | 品牌与更多操作 | `.brand-row` | 左侧 VTCC，右侧 44px 更多按钮 |
| 3 | 日期与日历入口 | `.date-row` | 日期、星期、辅助文案及日历按钮 |
| 4 | 搜索入口 | `.search` | 搜索赛事、战队或选手 |
| 5 | 今日比赛 | `[data-od-id="today-matches"]` | 支持正常、加载、空、错误四种状态 |
| 6 | 热门赛事 | `.events` | 横向滚动赛事卡片 |
| 7 | 近期赛程 | `.schedule-list` | 按日期分组的紧凑赛程列表 |
| 8 | 底部导航 | `.bottom` | 首页、赛事、收藏、我的；固定在底部 |
| 9 | 轻提示 | `.toast` | 页面操作后的短时反馈 |

## 2. 页面与模块尺寸

### 2.1 页面容器

- 基准宽度：`375px`。
- 页面最小高度：`100vh`。
- 主内容左右内边距：`16px`。
- 主内容底部预留：`88px`，避免被 72px 底部导航遮挡。
- 宽度达到 `600px` 后，内容最大宽度为 `430px` 并水平居中；微信手机端实现以 375px 视觉比例为主。
- 页面根容器横向溢出隐藏；内容不允许产生页面级横向滚动。

### 2.2 顶部区域

- 状态栏高度：`24px`；左右内边距 `4px`；底部内边距 `2px`。
- 顶部内容上/下内边距：`10px / 2px`。
- 品牌字号：`24px`，行高 `1`。
- 顶部图标按钮：`44×44px`，圆角 `10px`。
- 日期行距品牌行：`18px`。
- 星期距日期：`8px`。
- 日期下辅助文案上边距：`4px`。

### 2.3 搜索入口

- 宽度：内容区 100%。
- 高度：`46px`。
- 顶部间距：`16px`。
- 水平内边距：`14px`。
- 图标与输入内容间距：`10px`。
- 圆角：`12px`。
- 搜索结果反馈层：距搜索框 `8px`，内边距 `12px`。

### 2.4 通用模块标题

- 每个业务模块顶部间距：`20px`。
- 标题行高度：`28px`。
- 标题字号：`18px`。
- “全部”按钮右侧贴齐，内边距为 `10px 0 10px 12px`。

### 2.5 今日比赛

- 标题与比赛列表间距：`10px`。
- 比赛卡片之间间距：`10px`。
- 卡片内边距：`12px`。
- 卡片圆角：`12px`。
- 元信息行项目间距：`8px`。
- 战队对阵区距元信息行：`12px`。
- 对阵区三列：`1fr / 54px / 1fr`。
- 战队 Logo：`36×36px`，圆角 `9px`。
- Logo 与战队名称间距：`8px`。
- 比分区域最小宽度：`64px`。
- 比分数字使用等宽字体及 tabular numerals，强制单行；左右数字以冒号为中心保持等宽、对称。
- 比赛状态标签内边距：`3px 7px`，圆角 `6px`。

### 2.6 热门赛事

- 标题与横向区域间距：`10px`。
- 横向容器左右延伸至屏幕安全边缘：外边距 `0 -16px`、内边距 `0 16px 4px`。
- 卡片间距：`10px`。
- 单卡固定宽度：`108px`，不压缩。
- 卡片内边距：`12px`；圆角 `12px`。
- 赛事标识：`34×34px`，圆角 `9px`，下边距 `10px`。
- 横向滚动条隐藏，但必须保留手势滚动能力。

### 2.7 近期赛程

- 日期标签：上边距 `10px`、下边距 `7px`。
- 列表圆角：`12px`，内容裁切。
- 每行高度：`58px`。
- 行水平内边距：`12px`。
- 左侧赛事元信息固定宽度：`92px`。
- 右侧对阵元素间距：`9px`。
- 小型 Logo：`24×24px`，圆角 `7px`。

### 2.8 底部导航与提示

- 底部导航固定在视口底部，高度 `72px`，宽度 100%。
- 四个导航项等分，每项宽度 `25%`。
- 导航图标：`22×22px`；图标与文字间距 `4px`。
- Toast 距底部 `86px`，水平居中；内边距 `9px 14px`，圆角 `8px`。

## 4. 圆角、边框和阴影

### 圆角

- 主卡片、搜索框、结果层、赛事卡片、状态容器：`12px`。
- 顶部图标按钮：`10px`。
- 战队 Logo：`9px`。
- 重试按钮、小型赛事 Logo：`9px / 7px`。
- 比赛状态标签：`6px`。
- 骨架块：`5px`。
- 状态图标容器：圆形。

### 边框

- 搜索框、比赛卡、热门赛事卡、赛程列表、状态容器、重试按钮：`1px solid #E5E6EB`。
- 搜索聚焦：边框切换为 `#FF4655`，不改变尺寸。
- 赛程行：底部 `1px solid #E5E6EB`；最后一行无底边框。
- 底部导航：顶部 `1px solid #E5E6EB`。

### 阴影

- 搜索框：`0 2px 8px rgba(29, 33, 41, 0.025)`。
- 比赛卡：`0 2px 8px rgba(29, 33, 41, 0.035)`。
- 其他卡片不增加阴影。

## 5. 横向滚动区域

热门赛事是页面唯一业务横向滚动区：

- 容器使用横向排列，卡片禁止收缩。
- 首张卡与页面内容左边界对齐，末张卡保留 16px 安全间距。
- 隐藏可视滚动条，不禁用触摸滚动。
- 微信原生小程序建议使用横向 `scroll-view` 实现，保持 `show-scrollbar` 关闭。
- 不使用横向滚动承载今日比赛或近期赛程。

## 6. 页面交互

| 触发点 | 当前原型行为 | 微信原生实现要求 |
|---|---|---|
| 搜索框获得焦点 | 显示搜索反馈层 | 保持输入可编辑，展示搜索建议或反馈 |
| 输入搜索词 | 显示“正在搜索…”反馈 | 接入防抖搜索；无结果时使用真实空反馈 |
| 搜索框失焦 | 延迟约 180ms 收起反馈层 | 允许用户点击建议后再关闭 |
| 日历按钮 | Toast 提示入口已响应 | 打开日期选择器/日历组件 |
| 更多按钮 | Toast 提示入口已响应 | 打开业务定义的更多操作 |
| 比赛卡 | 点击反馈 | 跳转比赛详情；本交付不定义详情页 |
| 热门赛事卡 | 点击反馈 | 跳转对应赛事页；本交付不定义赛事页 |
| “全部” | 点击反馈 | 跳转对应列表页 |
| 收藏导航 | 点击反馈 | 切换至收藏 Tab |
| 无比赛“查看赛程” | 平滑滚动至近期赛程 | 原生实现滚动到近期赛程锚点 |
| 错误态“重试” | 加载约 700ms 后恢复正常态 | 重新请求数据，成功后渲染正常态 |
| Toast | 约 1500ms 后隐藏 | 同一时间只显示一条短反馈 |

比赛卡按压态缩放至 `0.99`，持续约 `150ms`。系统设置“减少动态效果”时，关闭动画和过渡。

## 7. 页面状态

首页通过今日比赛模块状态切换，其他模块及页面结构保持不变。

### 正常状态 `normal`

- 展示比赛列表。
- 每张卡必须包含：赛事名称、比赛时间、BO3/BO5、比赛状态、双方 Logo、双方名称、比分或 VS。
- 直播比分中的领先数字使用品牌强调色。

### 加载状态 `loading`

- 今日比赛区域替换为最小高度 `236px` 的状态容器。
- 使用结构化骨架：一条短标题骨架和两条 `70px` 高卡片骨架。
- 骨架动画周期约 `1.35s`。
- 显示“正在加载赛事数据”。
- 模块标题及后续模块不移动、不消失。

### 今日无比赛状态 `empty`

- 状态容器最小高度 `236px`，内边距 `24px`。
- 使用 `48×48px` 圆形日历图标容器。
- 标题：“今天暂无比赛”。
- 说明：“可以先看看明天的近期赛程”。
- 操作：“查看赛程”，点击后定位近期赛程模块。

### 加载失败状态 `error`

- 状态容器规格与空状态一致。
- 使用警示图标。
- 标题：“加载失败”。
- 说明：“网络似乎开了小差，请稍后重试”。
- 操作：“重试”；点击后先进入加载态，成功后恢复正常态。

## 8. 开发验收要点

- 首页视觉必须与 `vtcc-home.html` 一致，不额外添加设计控制、调试面板或平台切换控件。
- 375px 宽度下无页面级横向溢出；只有热门赛事区域可以横向滚动。
- 所有图标按钮实际点击区域不小于 `44×44px`。
- 底部导航固定且不遮挡最后一条赛程。
- 比分左右数字必须以冒号为中心等宽对称、保持单行。
- 正常、加载、无比赛、错误四种状态均可独立复现。
- 所有最终用户界面文字使用简体中文。
