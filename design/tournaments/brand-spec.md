# VTCC 微信原生小程序：品牌与基础令牌规范

> 本规范直接提取自最终确认的 `vtcc-home.html`。实现时不得自行替换颜色、字体、间距或圆角体系。

## 1. 品牌颜色

| 令牌 | 色值 | 用途 |
|---|---|---|
| `accent` | `#FF4655` | VTCC 品牌强调、直播状态、进行中状态、领先比分、当前底部导航、搜索聚焦边框 |
| `accent-soft` | `#FFF0F1` | 直播状态标签的浅色背景 |

品牌红只用于需要立即识别的状态与当前选择，不作为大面积背景，不用于普通正文或装饰。

## 2. 背景颜色

| 令牌 | 色值 | 用途 |
|---|---|---|
| `bg` | `#F7F8FA` | 页面背景、普通状态标签背景、事件标识浅底、状态图标浅底 |
| `surface` | `#FFFFFF` | 搜索框、卡片、列表、底部导航、状态容器、按钮表面 |

底部导航使用接近不透明白色：`color-mix(in oklch, #FFFFFF 98%, transparent)`；微信实现可采用 `rgba(255,255,255,0.98)`。

## 3. 文字颜色

| 令牌 | 色值 | 用途 |
|---|---|---|
| `fg` | `#1D2129` | 品牌、日期、模块标题、战队名称、比分、主要正文 |
| `muted` | `#4E5969` | 比赛元信息、普通状态、日期分组、VS |
| `subtle` | `#86909C` | 辅助文案、搜索占位、更多入口、赛程元信息、未激活导航、状态说明 |
| `surface` | `#FFFFFF` | 深色 Logo 或 Toast 上的反白文字 |

## 4. 状态与边框颜色

| 令牌/状态 | 色值 | 用途 |
|---|---|---|
| `border` | `#E5E6EB` | 卡片边框、输入框边框、列表分割线、底部导航分割线 |
| `success` | `#00A870` | 成功类反馈的保留状态色；当前首页主界面未直接展示 |
| 直播/进行中 | `#FF4655` + `#FFF0F1` | 红字、浅红背景 |
| 未开始/中性 | `#4E5969` + `#F7F8FA` | 灰字、浅灰背景 |
| 错误态 | `#86909C` 图标与说明 | 当前设计不新增大面积红色警示 |
| Toast | `#1D2129` 背景 + `#FFFFFF` 文字 | 短时操作反馈 |

战队 Logo 的示例底色由品牌色、主文字色及基础色混合生成；正式开发应优先使用真实战队 Logo 资源，不把示例底色当作战队品牌标准。

## 5. 字体体系

### 中文与通用正文

```text
-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif
```

- iOS/微信环境优先使用系统字体和苹方。
- 不下载额外展示字体。
- 页面标题与正文使用同一系统字体体系，通过字号和字重建立层级。

### 数字与比分

```text
"SFMono-Regular", Consolas, monospace
```

- 用于比分和 VS。
- 比分启用 tabular numerals。
- 左右数字必须以冒号为中心、等宽、单行、对称。

### 字重

| 字重 | 用途 |
|---:|---|
| 400 | 正文、辅助文字、元信息 |
| 650 | 状态栏、战队名、直播状态、当前导航 |
| 700 | 日期、热门赛事名称、比分、状态标题 |
| 750 | 模块标题 |
| 800 | VTCC 品牌、Logo 字母占位 |

微信端若不支持 650/750，可分别映射到 `600` 与 `700`，但必须保持原有视觉层级。

## 6. 间距体系

以 `4px` 为最小步进，主要采用 8px 基础节奏。

| 间距 | 用途 |
|---:|---|
| 2px | 顶部状态栏底部微调 |
| 4px | 辅助文案、图标与底部导航文字 |
| 7px | 日期分组与列表、状态说明 |
| 8px | 星期间距、元信息间距、Logo 与队名、搜索反馈层 |
| 9px | 近期赛程对阵元素间距、Toast 垂直内边距 |
| 10px | 卡片间距、横向卡片间距、标题与内容间距 |
| 12px | 卡片内边距、比赛信息层间距、状态图标间距 |
| 14px | 搜索框水平内边距、Toast 水平内边距 |
| 16px | 页面左右安全边距、搜索框顶部间距 |
| 18px | 品牌行与日期行间距 |
| 20px | 业务模块之间的垂直间距 |
| 24px | 空/错误/加载状态容器内边距 |
| 88px | 页面底部内容安全预留 |

全局基础间距令牌：`--space: 16px`。

## 7. 圆角体系

| 圆角 | 用途 |
|---:|---|
| 5px | 骨架块 |
| 6px | 比赛状态标签 |
| 7px | 近期赛程小 Logo |
| 8px | Toast；全局小圆角令牌 `r-sm` |
| 9px | 战队 Logo、重试按钮 |
| 10px | 顶部图标按钮 |
| 12px | 搜索框、比赛卡、热门赛事卡、赛程列表、状态容器；全局中圆角令牌 `r-md` |
| 50% | 状态图标圆形容器 |

## 8. 边框与阴影基线

- 标准边框：`1px solid #E5E6EB`。
- 搜索聚焦边框：`1px solid #FF4655`。
- 搜索框阴影：`0 2px 8px rgba(29,33,41,0.025)`。
- 比赛卡阴影：`0 2px 8px rgba(29,33,41,0.035)`。
- 不增加发光、霓虹、毛玻璃或大面积渐变。
