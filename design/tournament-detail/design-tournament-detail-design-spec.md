# VTCC 微信原生小程序：赛事详情页设计交付规范

> 基准稿：`vtcc-tournament-detail.html`  
> 基准设备：iPhone 13，390×844  
> 品牌来源：最终首页、赛事列表页和根目录 `brand-spec.md`；没有新增品牌令牌。

## 1. 页面结构

1. 47px iPhone 13 顶部安全区。
2. 44px 自定义导航栏：返回按钮、居中标题、右侧 94×32px 胶囊避让区。
3. 赛事信息卡：Logo、名称、简称、状态、阶段、日期和收藏。
4. 赛事概览：2×2 信息网格。
5. 比赛模块：状态筛选、日期分组和 MatchCard。
6. 参赛战队：横向滚动卡片。
7. 赛事介绍：简介、赛制、地点和时间。
8. 页面底部预留 49px 原生 TabBar 与 34px Home Indicator 安全区。

页面自然滚动，不设计复杂吸顶。

## 2. 画布与安全区

- 最大内容宽度：390px。
- 页面左右内边距：16px。
- 顶部安全区：47px。
- 自定义导航栏：44px。
- 原生 TabBar：49px；底部安全区：34px；额外内容间距：18px。
- 页面内容不得被原生 TabBar 遮挡。

## 3. 顶部导航

- 返回按钮：44×44px，实际图标 22×22px；左侧视觉补偿 -10px。
- 导航标题：16px / 700，水平居中。
- 胶囊避让：右侧 94×32px，不放置业务按钮。
- 页面滚动时导航不做复杂透明度或吸顶变化。

## 4. 赛事信息卡

- 距导航栏：12px。
- 内边距：16px。
- 圆角：12px。
- 边框：1px `#E5E6EB`。
- 阴影：`0 2px 8px rgba(29,33,41,0.035)`。
- 主布局：56px Logo / 自适应标题 / 44px 收藏按钮，列间距 12px。
- Logo：56×56px，圆角 12px。
- 标题：20px / 750 / 行高 1.25。
- 简称：10px 等宽字体，距标题 4px。
- 元信息：距顶部 14px，字号 11px，间距 7px。
- 收藏按钮：44×44px，圆角 10px；默认白底灰色，收藏后浅红底和品牌红图标。

## 5. 赛事概览

- 模块上边距：20px。
- 标题行高度：28px；标题 18px / 750。
- 2×2 网格，间距 10px。
- 单卡内边距：13px 12px；圆角 12px；1px 边框。
- 标签：11px，`#86909C`。
- 数值：15px / 700，距标签 5px。

## 6. 比赛模块

### 筛选

- 全部、即将开始、已结束。
- 胶囊高度 34px，水平内边距 14px，间距 8px。
- 选中态：`#FF4655` 背景、白色文字。
- 未选中：白色背景、深色文字、1px 浅灰内描边。
- 横向滚动，隐藏滚动条。

### MatchCard

- 日期分组标签：12px，距卡片 8px。
- 卡片内边距：12px；圆角 12px；卡间距 10px。
- 元信息：轮次、时间、BO 类型、状态，字号 11px。
- 对阵区距元信息 12px。
- 三列：`1fr / 64px / 1fr`。
- 战队 Logo：36×36px，圆角 9px。
- 战队名：14px / 650，单行省略。
- 比分：20px / 700，等宽数字；以冒号为中心保持单行对称。
- 获胜方比分使用品牌红。
- 按压态缩放至 0.99，约 150ms。

MatchCard 样式与首页已确认比赛卡保持一致。

## 7. 参赛战队

- 横向容器延伸到页面边缘，起止保留 16px 安全间距。
- 卡片固定宽度：96px；间距 10px。
- 卡片内边距：12px 10px；圆角 12px；1px 边框。
- Logo：40×40px，圆角 9px。
- 战队名：12px / 700；地区：10px。
- 每屏可看到约 3.5 支战队。

## 8. 赛事介绍

- 白色卡片，内边距 14px，圆角 12px，1px 边框。
- 简介：13px，行高 1.65，使用次要文字色。
- 信息行距简介 12px，并以 1px 分割线隔开。
- 行布局：64px 标签 + 自适应内容，字号 12px，间距 8px。

## 9. 页面状态

### 正常

- 展示完整赛事信息、概览、真实比赛结果、参赛战队和简介。

### Loading

- 保留导航栏。
- 赛事信息卡使用 Logo、标题和元信息骨架。
- 下方使用 3 个 96px 卡片骨架。
- 骨架动画周期 1.35s。

### 暂无比赛数据

- 保留赛事基本信息。
- 比赛模块显示日历图标、标题“暂无比赛数据”和同步说明。
- 不隐藏赛事名称与状态。

### 加载失败

- 导航栏下显示居中的错误容器。
- 警示图标、标题、说明与“重试”按钮。
- 重试后先显示 Loading，再恢复正常内容。

## 10. 交互

- 返回：回到赛事列表页。
- 收藏：切换收藏状态并本地持久化，显示 1500ms Toast。
- 比赛筛选：立即更新当前比赛列表。
- 比赛卡：进入比赛详情页。
- 战队卡：进入战队详情页。
- 下拉刷新：顶部下拉超过约 70px 触发。
- 支持减少动态效果偏好。

## 11. 数据字段

赛事：`id`、`name`、`abbr`、`region`、`status`、`stage`、`startDate`、`endDate`、`teamCount`、`matchCount`、`logo`、`description`、`format`、`location`。

比赛：`date`、`time`、`round`、`bestOf`、`status`、`teamA`、`teamB`、`scoreA`、`scoreB`。

战队：`id`、`name`、`abbr`、`regionOrCountry`、`logo`。

## 12. 验收标准

- 390×844 下无页面级横向溢出。
- 顶部返回栏避让微信胶囊。
- 内容不被原生 TabBar 和底部安全区遮挡。
- 第一屏可看到赛事名称、状态和主要信息。
- 比分数字以冒号为中心等宽对称。
- 四种页面状态可通过 `?state=normal|loading|empty|error` 复现。
- 所有界面文字为简体中文。
