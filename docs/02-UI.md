# VTCC UI Design Guidelines

## 1. 设计目标

VTCC 的 UI 以“简洁、清晰、高效查询”为核心。

本项目不采用重度暗黑电竞风，而是使用白色为主的轻量视觉风格，通过少量红色和深色元素体现 Valorant / VCT 的电竞属性。

设计关键词：

* 简洁
* 清晰
* 轻量
* 专业
* 信息优先
* 易于阅读

---

## 2. 设计原则

### 2.1 信息优先

VTCC 是赛事数据查询工具，不是内容社区。

页面设计应优先保证：

* 比赛时间清晰
* 对阵双方清晰
* 比分清晰
* 赛事状态清晰
* 页面跳转入口清晰

避免为了视觉效果降低信息可读性。

### 2.2 白色为主

大面积使用白色和浅灰色，保持页面轻盈。

深色仅用于：

* 主要文字
* 强调信息
* Logo
* 局部按钮
* 比赛比分

### 2.3 红色点缀

红色只用于关键状态和主要操作，不应大面积使用。

适用场景：

* 直播中
* 当前选中状态
* 收藏按钮
* 主按钮
* 重要比分
* 品牌标识

### 2.4 一致性

所有页面需要统一：

* 字体层级
* 圆角
* 页面边距
* 卡片间距
* 标签样式
* 按钮样式
* 图标风格

# 4. 字体规范

微信小程序使用系统默认字体，不额外加载字体文件。

推荐字体：

```css
font-family:
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

## 字号层级

| 类型    |    字号 |  字重 | 使用场景        |
| ----- | ----: | --: | ----------- |
| 页面主标题 | 40rpx | 600 | 首页标题、详情页标题  |
| 模块标题  | 34rpx | 600 | 今日比赛、热门赛事   |
| 卡片主标题 | 30rpx | 600 | 战队名、赛事名     |
| 正文    | 28rpx | 400 | 普通信息        |
| 辅助文字  | 24rpx | 400 | 时间、状态、说明    |
| 小标签   | 22rpx | 500 | 赛区、BO3、比赛状态 |
| 核心比分  | 48rpx | 700 | 比赛详情比分      |

文字行高建议为字号的 `1.4–1.6` 倍。

---

# 5. 间距规范

采用 `8rpx` 为基础单位。

推荐间距：

| 类型     |    数值 |
| ------ | ----: |
| 页面左右边距 | 32rpx |
| 页面顶部间距 | 24rpx |
| 模块之间   | 40rpx |
| 卡片之间   | 24rpx |
| 卡片内边距  | 28rpx |
| 元素小间距  |  8rpx |
| 元素常规间距 | 16rpx |
| 元素大间距  | 24rpx |

原则：

* 同一模块内元素间距较小
* 不同模块之间间距较大
* 页面不要显得拥挤
* 信息不能离得过散

---

# 6. 圆角与阴影

## 6.1 圆角

| 元素   |    圆角 |
| ---- | ----: |
| 大卡片  | 24rpx |
| 普通卡片 | 20rpx |
| 搜索框  | 20rpx |
| 按钮   | 16rpx |
| 标签   |  8rpx |
| 圆形头像 |   50% |

整体圆角保持克制，不采用过度圆润的风格。

## 6.2 阴影

卡片优先通过背景色、间距和边框区分。

只使用轻微阴影：

```css
box-shadow: 0 8rpx 24rpx rgba(29, 33, 41, 0.06);
```

避免：

* 浓重阴影
* 多层阴影
* 发光效果
* 霓虹效果

---

# 7. 卡片规范

## 7.1 比赛卡片

比赛卡片应包含：

* 比赛状态
* 比赛时间
* 所属赛事
* 战队 A 名称和 Logo
* 战队 B 名称和 Logo
* 比分或“VS”
* BO 类型
* 点击反馈

推荐结构：

```text
┌─────────────────────────────┐
│ VCT Pacific      今天 18:00 │
│                             │
│ [Logo] Gen.G     2 : 1 PRX  │
│                       [Logo]│
│                             │
│ 已结束                 BO3  │
└─────────────────────────────┘
```

设计要求：

* 比分是卡片视觉重点
* 战队名称最多显示一行
* Logo 尺寸统一
* 状态使用小标签
* 卡片整体可点击
* 点击后有轻微按压反馈

---

## 7.2 赛事卡片

赛事卡片应包含：

* 赛事 Logo
* 赛事名称
* 赛区
* 当前阶段
* 开始和结束时间
* 赛事状态

推荐结构：

```text
┌─────────────────────────────┐
│ [Logo] VCT Masters Toronto  │
│        国际赛事 · 淘汰赛    │
│        06.07 - 06.22        │
│                    进行中 > │
└─────────────────────────────┘
```

---

## 7.3 战队卡片

战队卡片包含：

* 战队 Logo
* 战队名称
* 所属赛区
* 简短状态或排名
* 跳转箭头

---

## 7.4 选手卡片

选手卡片包含：

* 选手头像
* 游戏 ID
* 真实姓名或国籍
* 所属战队
* 位置或角色

背景：


文字：

```css
#1D2129
```

* 筛选
* 取消

# 9. 标签规范

标签主要用于：

* 比赛状态
* 赛事阶段
* 所属赛区
* BO 类型
* 战队地区
* 选手角色

标签建议：

```text
高度：40rpx
左右内边距：12rpx
圆角：8rpx
字号：22rpx
```

避免同一张卡片出现过多标签。

每张卡片建议不超过 3 个标签。

---

# 10. 图标规范

优先使用统一的线性图标。

图标风格：

* 线条简洁
* 粗细统一
* 不使用写实图标
* 不混用多套图标
* 常规颜色为深灰
* 选中颜色为品牌红

常用图标：

* 首页
* 赛事
* 收藏
* 我的
* 搜索
* 返回
* 筛选
* 时间
* 地区
* 分享
* 右箭头
* 空状态
* 错误提示

---

# 11. TabBar 规范

TabBar 包含四项：

1. 首页
2. 赛事
3. 收藏
4. 我的

样式：

* 背景：白色
* 顶部分割线：浅灰色
* 未选中图标：灰色
* 未选中文字：灰色
* 选中图标：品牌红
* 选中文字：品牌红

不使用过度复杂的自定义 TabBar，MVP 优先使用微信原生 TabBar。

---

# 12. 搜索框规范

搜索框使用浅灰背景，保持轻量。

推荐样式：

```text
高度：80rpx
背景：#F2F3F5
圆角：20rpx
左右内边距：24rpx
搜索图标：灰色
占位文字：#86909C
```

占位文案：

```text
搜索赛事、战队或选手
```

搜索框点击后跳转到独立搜索页。

---

# 13. 页面状态规范

每个数据页面必须考虑四种状态：

## 13.1 加载状态

* 使用骨架屏或简单 Loading
* 不长时间显示空白页
* 加载文字保持简短

推荐文案：

```text
正在加载赛事数据
```

## 13.2 空状态

包含：

* 简洁图标
* 状态标题
* 一句说明
* 可选操作按钮

示例：

```text
今天暂无比赛

可以看看近期赛事或关注其他赛区
```

## 13.3 错误状态

包含：

* 错误图标
* 简短说明
* 重试按钮

示例：

```text
加载失败

请检查网络后重试
```

## 13.4 正常状态

数据内容按视觉层级正常展示。

---

# 14. 页面设计规范

## 14.1 首页

首页模块顺序：

1. 页面标题与日期
2. 搜索入口
3. 今日比赛
4. 热门赛事
5. 可选的近期比赛

首页不使用大面积 Banner。

首页应让用户打开后立即看到今天的比赛。

## 14.2 赛事列表页

页面包含：

* 页面标题
* 赛区筛选
* 状态筛选
* 赛事卡片列表

筛选使用横向滚动胶囊按钮。

## 14.3 比赛详情页

视觉重点：

1. 比赛状态
2. 对阵战队
3. 比分
4. 比赛时间
5. 分地图比分
6. 出场阵容

比分区域可以使用轻微浅灰背景或浅红背景突出，但不能大面积使用红色。

## 14.4 战队详情页

页面结构：

1. 战队基础信息
2. 当前阵容
3. 近期比赛
4. 参加赛事

## 14.5 选手详情页

页面结构：

1. 头像与基本信息
2. 所属战队
3. 位置或角色
4. 基础数据
5. 近期比赛

## 14.6 收藏页

使用顶部分类切换：

* 赛事
* 战队
* 选手

空状态需要明确引导用户去赛事页或搜索页。

# 15. 交互规范

所有可点击区域必须：

* 有明确点击范围
* 有视觉反馈
* 不依赖过小图标
* 点击区域不小于 `80rpx × 80rpx`

页面跳转规则：

* Tab 页面使用 `switchTab`
* 普通详情页使用 `navigateTo`
* 返回上一页使用系统返回
* 同一页面避免重复跳转
* 跳转时必须传递明确 ID

---

# 16. 图片规范

战队 Logo、赛事 Logo 和选手头像需要统一尺寸。

建议：

| 图片类型        |     尺寸 |
| ----------- | -----: |
| 比赛卡片战队 Logo |  64rpx |
| 战队详情 Logo   | 144rpx |
| 赛事卡片 Logo   |  80rpx |
| 选手列表头像      |  72rpx |
| 选手详情头像      | 160rpx |

图片加载失败时必须显示默认占位图。

避免图片变形，统一使用：

```css
mode="aspectFit"
```

头像照片可根据实际情况使用：

```css
mode="aspectFill"
```

---

# 17. 无障碍与可读性

* 主要正文不小于 `28rpx`
* 辅助文字不小于 `22rpx`
* 不使用浅灰文字表达关键信息
* 红色不能作为唯一的状态区分方式
* 可点击文字必须有足够点击区域
* 比分、时间和状态需要有明确文字说明

---

# 18. 设计限制

MVP 阶段不使用：

* 复杂动画
* 视频背景
* 大面积渐变
* 霓虹发光
* 毛玻璃效果
* 3D 元素
* 过度装饰
* 复杂自定义导航栏
* 大面积 Valorant 游戏素材

目的是保证：

* 开发成本可控
* 页面加载稳定
* 信息清晰
* 作品集观感专业

---

# 19. 首页视觉方向

首页推荐视觉效果：

```text
白色卡片
+
浅灰页面背景
+
深色文字
+
少量品牌红
+
清晰的赛事信息
```

最终观感应接近：

* 体育赛事查询工具
* 简洁的数据产品
* 现代移动端产品
* 轻量电竞视觉

而不是：

* 游戏启动页
* 电竞直播平台
* 暗黑资讯社区
* 高饱和宣传海报
